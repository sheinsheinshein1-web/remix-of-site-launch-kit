# Картинки

## Правило

Все картинки в `src/assets/` должны быть в формате **WebP** (кроме SVG-логотипов).
WebP в 2–3 раза легче PNG/JPG — это критично для скорости сайта на мобильном.

## Как добавить новую картинку

1. Положи файл (PNG / JPG / WebP) в `src/assets/` или в подпапку производителя (например, `src/assets/glezman/`).
2. Запусти команду:

   ```bash
   node scripts/optimize-images.mjs
   ```

3. Скрипт сам:
   - конвертирует все PNG/JPG в WebP (q=75, визуально идентично)
   - удалит оригиналы
   - обновит все импорты в коде

4. Закоммить результат в Git.

## Что нельзя

- Не коммить тяжёлые `.png` или `.jpg` напрямую — сайт замедлится.
- Не трогай файлы `logo-*.svg`, `logo-*.png`, `share-icon.*` — они в исключениях.

## Оптимизации, которые работают автоматически

- **Lazy loading**: все слайды в `SwipeableGallery` (карточки) — `loading="lazy" decoding="async"`,
  кроме первого (LCP).
- **Hero preload**: `<link rel="preload" as="image">` на `cat-houses.webp` в `HeroSection`.
- **Single source of truth**: добавил проект в `src/data/projects.ts` —
  карточка автоматически попадает в каталог, ленту, поиск, избранное и партнёрскую страницу.
